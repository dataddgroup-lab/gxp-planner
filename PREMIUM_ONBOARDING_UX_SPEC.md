## DESIGN SYSTEM: TRUST + POWER

### Color Palette

**Primary (Trust & Authority):**
- Deep Navy: #001F3F (primary CTA, headings)
- Warm White: #F8F9FA (clean background)

**Accent (Capability & Progress):**
- Clinical Green: #00AA44 (completion, validation passed)
- Warm Gold: #FFA500 (attention, warning, next action)
- Alert Red: #CC0000 (blockers, serious issues)

**Neutrals (Precision):**
- Dark Gray: #333333 (body text, secondary info)
- Light Gray: #CCCCCC (borders, dividers, disabled states)

**Why this palette:**
- Navy = pharmaceutical authority (FDA, regulated industry)
- Gold = professional premium (not bright, but confident)
- Green = validation/compliance (passed)
- No bright colors (maintains trust, not toy-like)

### Typography

**Headings:** Inter Bold, 24–32px (modern, clean, authoritative)
**Body:** Inter Regular, 14–16px (readable, professional)
**Monospace (for technical data):** JetBrains Mono, 12–14px (precision, clarity)

**Why Inter:** Used by Apple, Stripe, Vercel. Professional, trustworthy, modern.

### Spacing & Layout

**8px grid system** (multiples of 8: 8, 16, 24, 32, 40, 48...)
- Consistent, predictable, professional
- Reduces cognitive load (everything is aligned)
- Scales from mobile to desktop smoothly

**Card design:**
- 16px rounded corners (modern, not too soft)
- 2px border, light gray (#CCCCCC)
- Subtle shadow (0 4px 8px rgba(0,0,0,0.08))
- White background on light gray canvas

## ANIMATION GUIDELINES: WOW WITHOUT DISTRACTION

### Principles

1. **Purpose First**
   - Every animation serves a purpose (feedback, transition, education)
   - No animation without reason
   - Remove after MVP if not essential

2. **Speed: 300–600ms Standard**
   - 300ms: Micro-interactions (button press, field validation)
   - 400–500ms: Transitions (screen to screen)
   - 600–800ms: Entrance animations (page load)
   - >1s: Only for critical teaching moments (onboarding)

3. **Easing: Subtle, Not Bouncy**
   - cubic-bezier(0.25, 0.1, 0.25, 1.0) ← standard easing
   - cubic-bezier(0.34, 1.56, 0.64, 1) ← subtle bounce (sparingly)
   - NO spring animations (unprofessional in regulated context)

4. **Performance**
   - GPU-accelerated (transform, opacity only)
   - NO layout shifts (no reflow during animation)
   - Target 60fps (use DevTools to verify)
   - Mobile should be smooth too

### Specific Animation Examples

**Button Press:**
```css
.button {
  transition: all 200ms cubic-bezier(0.25, 0.1, 0.25, 1.0);
}
.button:active {
  transform: scale(0.98);
  opacity: 0.8;
}
```
(Subtle, responsive, confident)

**Form Field Validation:**
```css
.input-valid::after {
  content: "✓";
  animation: slideInScale 300ms cubic-bezier(0.25, 0.1, 0.25, 1.0);
}
@keyframes slideInScale {
  from { opacity: 0; transform: scale(0.5); }
  to { opacity: 1; transform: scale(1); }
}
```

**Page Transition:**
```css
.page-enter {
  animation: fadeInUp 400ms cubic-bezier(0.25, 0.1, 0.25, 1.0);
}
@keyframes fadeInUp {
  from { opacity: 0; transform: translateY(20px); }
  to { opacity: 1; transform: translateY(0); }
}
```
